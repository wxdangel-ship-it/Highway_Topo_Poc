# 架构决策

## 状态

- 草案状态：Round 1 占位文档
- 用途：在项目级和模块级文档迁移正式开始后，存放长期有效的决策记录

## Round 1 说明

Round 1 不会建立完整的 ADR 集合。本轮只建立落位目录，并记录当前已经显现、后续应被正式决策化的主题，包括：

- T05-V2 相对 legacy T05 的模块定位
- `t03` 作为缺失 taxonomy 成员的处理方式
- `t10` 作为额外模块且存在命名漂移的处理方式
- `AGENTS.md` 与 `SKILL.md` 的最终收口规则

## 后续预期用法

未来的决策记录至少应包含：

- 决策本身
- 背景与备选方案
- 后果
- 受影响的模块与文档面
