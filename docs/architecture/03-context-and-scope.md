# 03 上下文与范围

## 状态

- 草案状态：Round 1 最小可信草案
- 来源依据：
  - `SPEC.md`
  - `docs/codebase-research.md`
  - 当前 `modules/`、`src/` 与 `tests/` 布局
- 审核重点：
  - 确认模块家族边界
  - 确认项目上下文抽象层级是否合适

## 系统上下文

该仓库处于两个执行环境之间：

- 外网工程环境：
  - 源码与文档编写
  - 版本控制
  - 合成回归与分析
- 内网执行环境：
  - 真实 patch 数据
  - 模块运行
  - 文本 bundle 反馈

## 当前功能范围

当前项目范围覆盖：

- 合成数据与 QC 支撑（`t00`、`t01`、`t02`）
- 锚点生成（`t04`）
- 道路拓扑生成（`t05`、`t05-V2`）
- patch 级预处理 / 后处理（`t06`、`t07`）
- 额外的路口建模工作（`t10`）

## 文档范围

当前文档系统包括：

- 全局范围与协议文档
- 模块契约与模块规则
- 审核与验收说明
- 审计与运行级材料
- 新接入的 spec-kit 工作流资产

Round 1 的重点是梳理这些文档类型之间的关系，而不是一次性重写全部文档正文。

## Round 1 非范围

- 全量 legacy 文档迁移
- 运行时行为修改
- 算法或接口重设计
- 除显式记录外，对模块 taxonomy 做结构归一化
