# CodeX Guardrails (Global, MUST FOLLOW)

## 0. 总原则：Stop → Ask → Do
- 任何不清晰的地方，优先提出问题给用户确认
- 未确认前，不进行大规模落地（最多允许创建空目录与占位 README）

## 1. 必须先问的触发条件（示例）
- 目录结构、模块命名、运行入口、schema 字段、输出格式
- 任何“可能影响内网回传文本”的格式变更
- 任何涉及 RC/SW/锚点定义的细节（留给子 Agent）

## 2. 外传文本硬约束（禁止项）
外传文本（将被用户粘贴回外网）中严禁出现：
- 任意坐标（x/y/z/lat/lon/utm/epsg 等）
- 任意几何顶点数组（polyline/polygon 坐标序列）
- 任意内网路径、机器信息、用户名
- 任意大段原始序列/逐帧明细（超过 Top-K）

## 3. 允许的“问题定位”方式
- 使用“索引化位置/区间”（bin 区间），不使用坐标
- 使用统计摘要（p50/p90/p99）+ 阈值 + 严重程度等级

## 4. 执行前的标准输出（握手）
在开始写代码前，必须输出：
- 理解摘要（<=15行）
- 待确认问题（<=5条）
- 最小落地计划（<=10步）
等待用户确认后再执行
# Workspace Setup (WSL + Python)

## 1. 路径硬约束
- 本项目必须放在 Windows 的 E: 盘下
- WSL 下对应路径通常为：/mnt/e/...
- 推荐项目根目录：
  /mnt/e/Highway_Topo_Poc/

## 2. 路径相关注意事项
- 代码与配置尽量使用相对路径
- 外传文本中禁止出现任何本机绝对路径
- 若需要记录文件名，仅记录相对路径或逻辑名（不含盘符/挂载路径）

## 3. Python 环境
- 使用 WSL 的 Python（版本与依赖管理方式由实现者结合本机情况选择）
- 需要可一键运行（脚本/Makefile/命令均可）
