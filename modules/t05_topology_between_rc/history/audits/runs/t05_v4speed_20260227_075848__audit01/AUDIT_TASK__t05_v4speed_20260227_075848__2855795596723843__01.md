# CodeX 任务书（t05 / 审计模式，不改代码）
> 目的：外网 CodeX 对现有实现做“只读审计 + 证据解释 + 最小修复建议”，不提交任何代码改动。

## 输入（由用户提供给你）
- 文件：T05 内网问题证据简报（T05_AUDIT_BRIEF.md）

## 约束
- 禁止修改代码、禁止提交
- 允许运行 `python -c` / `pytest -q` / `rg` / `cat` / `sed` 等只读命令
- 输出必须包含行号定位与调用链解释

## 你需要完成的工作
1) 阅读 T05_AUDIT_BRIEF.md，理解冻结规则与证据
2) 在 repo 中定位以下实现点（必须给出文件:行号）：
   - GeoJSON CRS 解析/校验/归一化（CRS84/URN/EPSG）
   - Vector/DivStripZone.geojson 的读取与重投影是否发生（是否统一到 EPSG:3857）
   - traj_surface 构建：slice、P02/P98、polygon 拼接/闭合、buffer、扣除 divstrip
   - traj_surface debug dump：为何落盘为 MultiLineString（不是 Polygon），落盘对象与 gate 对象是否一致
   - in_ratio 计算与 endpoint_in_surface 判定：使用的几何对象是什么
   - seg_index=0 长跳连：road 生成流程中是否缺少 endpoint 投影 substring 裁剪；或端点片段与 lb_path 不一致导致跳连
   - multi-road 通道一致性：是否 per-cluster 生成 surface/path/road 并择优；若不是，指出错配点
3) 复现/解释：
   - 为什么 DivStripZone CRS84 没被统一（或被错误当作 3857）
   - 为什么 traj_surface_best 不是 polygon
   - 为什么 debug surface 与 gate 指标矛盾（若存在）
4) 输出报告（不改代码）：
   - T05_AUDIT_REPORT.md（必须包含：结论、证据、最小修复建议、风险与回归点）
   - audit_trace_map.json（可选但建议）

## 输出模板
- Executive Summary（Top3 根因）
- Evidence table（每条证据对应代码位置）
- Root causes（按优先级）
- Minimal fix plan（只列策略，不写实现代码）
- Regression checklist（3 patch 回归要点）
